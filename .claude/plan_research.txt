## 2. LangGraph StateGraph Design

### 2.1 State Schema

The spec (§7.3) defines a TypeScript `Annotation.Root(...)`. The Python equivalent uses `TypedDict` with LangGraph's `add_messages` reducer for the messages field.

```python
# orchestrator/models/run_state.py

from __future__ import annotations
import time
from dataclasses import dataclass, field
from typing import Annotated, Any, Literal, Optional
from langchain_core.messages import BaseMessage
from langgraph.graph.message import add_messages


RunStatus = Literal[
    "running", "passed", "failed", "errored", "timed_out", "blocked"
]


@dataclass
class ErrorInfo:
    error_type: str          # llm_timeout | llm_rate_limit | tool_error | browser_crash | mcp_disconnect
    message: str
    step_number: int
    tool_name: Optional[str] = None
    recoverable: bool = True
    timestamp: float = field(default_factory=time.time)


@dataclass
class StepTrace:
    step_number: int
    timestamp: float
    completed_at: Optional[float] = None
    checkpoint_id: Optional[str] = None
    checkpoint_completed: bool = False
    # LLM fields
    llm_request_id: Optional[str] = None
    model: str = ""
    input_tokens: int = 0
    output_tokens: int = 0
    cache_read_tokens: int = 0
    cache_creation_tokens: int = 0
    llm_latency_ms: int = 0
    tool_execution_latency_ms: int = 0
    wait_for_stable_ms: int = 0
    total_step_latency_ms: int = 0
    # Tool fields
    tool_name: Optional[str] = None
    tool_source: Optional[str] = None   # "mcp" | "custom"
    tool_input: Optional[dict] = None
    tool_output: Optional[Any] = None
    tool_success: bool = True
    tool_error: Optional[str] = None
    tool_retries: int = 0
    # Agent text
    reasoning: Optional[str] = None
    agent_output_text: Optional[str] = None
    stop_reason: Optional[str] = None
    # Page context
    page_url: str = ""
    page_title: str = ""
    # Cost
    estimated_cost_usd: float = 0.0


class AgentState(TypedDict):
    # Test case context (set once at start)
    test_case: TestCase
    run_id: str
    container_host: str
    mcp_url: str
    cdp_url: Optional[str]

    # Conversation history (accumulates via add_messages reducer)
    messages: Annotated[list[BaseMessage], add_messages]

    # Step tracking
    step_number: int
    current_checkpoint: Optional[str]
    completed_checkpoints: list[str]

    # Page state
    current_url: str
    page_title: str
    page_snapshot: str

    # Tool execution
    last_tool_name: Optional[str]
    last_tool_result: Any
    last_tool_success: bool

    # Error tracking
    consecutive_errors: int
    last_error: Optional[ErrorInfo]
    error_history: list[ErrorInfo]

    # Per-step trace accumulator
    traces: list[StepTrace]
    current_trace: Optional[StepTrace]

    # Termination
    status: RunStatus
    goal_complete: bool
    termination_reason: Optional[str]

    # Runtime clock
    run_start_time: float
```

### 2.2 Graph Construction

```python
# orchestrator/agent/graph.py

from langgraph.graph import StateGraph, START, END
from orchestrator.models.run_state import AgentState
from orchestrator.agent.nodes import (
    observe_node, reason_node, act_node,
    goal_check_node, error_handler_node, finalize_node,
)
from orchestrator.agent.edges import (
    route_after_reason, route_after_act,
    route_after_goal_check, route_after_error,
)


def build_graph() -> StateGraph:
    builder = StateGraph(AgentState)

    builder.add_node("observe",        observe_node)
    builder.add_node("reason",         reason_node)
    builder.add_node("act",            act_node)
    builder.add_node("goal_check",     goal_check_node)
    builder.add_node("error_handler",  error_handler_node)
    builder.add_node("finalize",       finalize_node)

    builder.add_edge(START, "observe")
    builder.add_edge("observe", "reason")

    builder.add_conditional_edges(
        "reason",
        route_after_reason,
        {"act": "act", "goal_check": "goal_check", "error": "error_handler"},
    )

    builder.add_conditional_edges(
        "act",
        route_after_act,
        {"observe": "observe", "error": "error_handler"},
    )

    builder.add_conditional_edges(
        "goal_check",
        route_after_goal_check,
        {"finalize": "finalize", "observe": "observe"},
    )

    builder.add_conditional_edges(
        "error_handler",
        route_after_error,
        {"observe": "observe", "finalize": "finalize"},
    )

    builder.add_edge("finalize", END)

    return builder.compile()
```

Note: Phase 1 omits the `guard_rails` node (spec §7.4 navigation policy enforcement is a Phase 2 concern per CLAUDE.md). The spec's full graph includes it, but CLAUDE.md Phase 1 work items do not mention guard rails.

### 2.3 Node Specifications

**OBSERVE node** (`observe.py`):

```python
async def observe_node(state: AgentState) -> dict:
    """
    Inputs consumed:  mcp_url, cdp_url, test_case.constraints, run_start_time, step_number, traces
    Outputs produced: step_number, current_url, page_title, page_snapshot,
                      current_trace (partial), status (if timeout exceeded)

    Raises: Never — catches all exceptions and sets status=errored if unrecoverable.
    """
```

Behavior:
1. Check wall-clock timeout: if `time.time() - state["run_start_time"] > test_case.constraints.timeout_seconds`, return `{"status": "timed_out", "termination_reason": "wall_clock_timeout"}`.
2. Check step count: if `state["step_number"] >= test_case.constraints.max_steps`, return `{"status": "timed_out", "termination_reason": "max_steps_exceeded"}`.
3. Increment `step_number`.
4. Create a new `StepTrace(step_number=..., timestamp=time.time())`.
5. Call `wait_for_stable` custom tool with a timeout of 8 seconds. Record `wait_for_stable_ms`.
6. Call `browser_snapshot` via MCP client. Parse the result to extract `current_url`, `page_title`, `page_snapshot` (the accessibility tree text). On MCP failure, retry up to 5 times with 1s delay (per spec §7.3 `retry: { maxAttempts: 5 }`). After 5 failures, set `status: errored`.
7. Return updated state fields plus partial `current_trace`.

**REASON node** (`reason.py`):

```python
async def reason_node(state: AgentState) -> dict:
    """
    Inputs consumed:  test_case, messages, page_snapshot, current_url, page_title,
                      completed_checkpoints, current_checkpoint, step_number
    Outputs produced: messages (appended with AIMessage), current_trace (with LLM fields)

    Retry policy: 3 attempts with exponential backoff (2s, 4s, 8s) for rate limit / server error.
    Fatal: 401, model not found — returns status=errored immediately.
    """
```

Behavior:
1. Build the merged tool list: `mcp_tools + custom_tool_schemas`.
2. Construct the user turn message: inject current page snapshot, current URL/title, and active checkpoint hint.
3. Call `anthropic.messages.create(...)` with streaming enabled. Record `time_to_first_token_ms`.
4. Extract `usage.input_tokens`, `usage.output_tokens`, `usage.cache_read_input_tokens`, `usage.cache_creation_input_tokens`. Extract `x-request-id` header.
5. Extract `thinking` blocks (reasoning) and `text` blocks (agent_output_text).
6. Compute `estimated_cost_usd` using the model pricing table.
7. Append the LLM response as `AIMessage` to `messages`.
8. Update `current_trace` with all LLM fields.
9. For rate limit (429): wait with exponential backoff (2s → 4s → 8s), retry up to 3 times. After 3 failures, set `last_error` with `error_type="llm_rate_limit"` and route to error_handler.
10. For auth error (401) or unknown model: immediately set `status: errored`, `termination_reason: "llm_auth_error"`.

**ACT node** (`act.py`):

```python
async def act_node(state: AgentState) -> dict:
    """
    Inputs consumed:  messages (last AIMessage with tool_use content), mcp_url, test_case
    Outputs produced: messages (appended with ToolMessage), last_tool_name, last_tool_result,
                      last_tool_success, current_trace (with tool fields), consecutive_errors

    Retry policy: 2 attempts with 500ms delay for tool execution failures.
    """
```

Behavior:
1. Extract the `tool_use` block from the last `AIMessage`.
2. Route based on tool name:
   - `browser_*` tools: call MCP client `call_tool(name, input)`. `tool_source = "mcp"`.
   - All other tools: call the function from `tool_registry`. `tool_source = "custom"`.
3. Measure `tool_execution_latency_ms`.
4. On success: reset `consecutive_errors` to 0, append `ToolMessage` to `messages`, update `current_trace`.
5. On failure: increment `consecutive_errors`, record `tool_error`, append an error `ToolMessage` (so agent sees the error), update `current_trace` with `tool_success=False`.
6. On `consecutive_errors >= 3`: set `last_error` with `error_type="tool_error"`, `recoverable=False`.
7. Detect repetitive actions: if the last 5 `tool_name` values in traces are identical, inject a meta-prompt into messages: "You've attempted this action 5 times without progress. Try a different approach."
8. Append completed trace to `state["traces"]`.

**GOAL_CHECK node** (`goal_check.py`):

```python
def goal_check_node(state: AgentState) -> dict:
    """
    Inputs consumed:  messages (last AIMessage), completed_checkpoints, test_case
    Outputs produced: goal_complete, status, termination_reason, completed_checkpoints
    Pure function — no I/O. Routes based on text signals in LLM output.
    """
```

Behavior:
1. Scan `agent_output_text` in `current_trace` for `<GOAL_COMPLETE>` or `<GOAL_BLOCKED>`.
2. Also scan for `[S1 complete]`, `[S2 complete]`, etc. — update `completed_checkpoints`.
3. If `<GOAL_COMPLETE>`: set `goal_complete=True`, `status="passed"`.
4. If `<GOAL_BLOCKED>`: set `goal_complete=True`, `status="blocked"`, `termination_reason=<agent explanation>`.
5. If `status` is already `timed_out` or `errored`: set `goal_complete=True` to force finalize.
6. In `strict` mode: if a checkpoint is marked failed, set `status="failed"`.

**ERROR_HANDLER node** (`error_handler.py`):

```python
def error_handler_node(state: AgentState) -> dict:
    """
    Inputs consumed:  last_error, consecutive_errors, error_history
    Outputs produced: error_history (appended), status (if fatal), messages (injected recovery prompt)
    Classifies errors and decides recoverable vs fatal per spec §7.5 table.
    """
```

Behavior per spec §7.5 failure taxonomy:
- `llm_rate_limit`: already handled with backoff in `reason` node. If it reaches error_handler, it's been retried — route to observe (the wait was already done).
- `tool_error` with `consecutive_errors < 3`: inject error context into messages ("The last tool call failed: {error}. Adapt your approach."). Append to `error_history`. Route to `observe`.
- `tool_error` with `consecutive_errors >= 3`: set `status="errored"`, `termination_reason="max_consecutive_tool_errors"`.
- `mcp_disconnect`: attempt MCP client reconnection (up to 3 times). If reconnects: inject "MCP reconnected" message, route to `observe`. If not: `status="errored"`.
- `llm_auth_error` or any `recoverable=False` error: `status="errored"`.

**FINALIZE node** (`finalize.py`):

```python
async def finalize_node(state: AgentState) -> dict:
    """
    Inputs consumed:  traces, status, test_case, messages, page_snapshot
    Outputs produced: status (final), termination_reason
    Side effects:     Runs assertion engine, prints trace summary to stdout.
    """
```

Behavior:
1. Run the assertion engine against `state["traces"]` and current page snapshot.
2. If any assertion fails: upgrade `status` from `"passed"` to `"failed"` (if it was `passed`).
3. Compute and print run trace summary (tokens, latency, cost per step).
4. Return final `status`.

```python
def route_after_reason(state: AgentState) -> Literal["act", "goal_check", "error"]:
```

## 3. Tool Architecture

### 3.1 MCP Client (`orchestrator/mcp/client.py`)

The existing `scripts/test_mcp_wiring.py` uses `McpStdioClient` via subprocess spawning of `npx @playwright/mcp`. For Phase 1, the sandbox exposes Playwright MCP on an HTTP port (`:3100`). The spec (§5.1) says "Connect to Playwright MCP (container) via http://run-{id}:3100/mcp". The current sandbox uses host-mapped ports.

```python
class PlaywrightMcpClient:
    """
    HTTP Streamable MCP client connecting to @playwright/mcp over HTTP.

    Supports two transport modes:
    - http: connects to http://{host}:{port}/mcp  (preferred for containerized sandbox)
    - stdio: spawns npx @playwright/mcp --cdp-endpoint {cdp_url} as a subprocess
             (fallback for local dev where container's MCP port isn't HTTP-exposed)

    Phase 1 uses stdio transport to connect via the existing SandboxHandle.cdp_url,
    mirroring the proven pattern from test_mcp_wiring.py.
    """

    def __init__(self, transport: Literal["http", "stdio"], *, url: str | None = None, cdp_url: str | None = None) -> None: ...

    async def initialize(self) -> None:
        """Perform MCP handshake, populate self._tools."""

    async def list_tools(self) -> list[McpToolSchema]: ...

    async def call_tool(self, name: str, arguments: dict) -> McpToolResult:
        """
        Returns McpToolResult(content: str, is_error: bool).
        Raises McpToolError on timeout or connection failure.
        Retries up to 2 times with 500ms delay on connection errors.
        """

    async def close(self) -> None: ...
```

Design decision: Phase 1 uses `stdio` transport (spawning `npx @playwright/mcp --cdp-endpoint {cdp_url}`) because the existing sandbox doesn't expose port 3100 over HTTP. This matches the proven pattern in `test_mcp_wiring.py`. Phase 3 switches to HTTP transport when the Docker bridge network is implemented.

### 3.3 Custom Tools

Each custom tool is a Python async function with a type-annotated signature. They are registered in `tool_registry.py` and exposed to the LLM via JSON Schema definitions in `schemas.py`.

#### `wait_for_stable` (`tools/wait_for_stable.py`)

The spec (§6.2) defines four signals. Phase 1 implements two (network quiescence + DOM settle via CDP), with visual stability deferred to Phase 2.

```python
async def wait_for_stable(
    *,
    cdp_session: CdpSession,
    page_type: str = "spa",
    timeout_ms: int = 8000,
    network_idle_ms: int = 300,
    dom_settle_ms: int = 500,
) -> WaitForStableResult:
    """
    Phase 1 implementation — two signals:
    1. Network quiescence: 0 pending XHR/fetch for network_idle_ms.
       Tracks via CDP Network domain events buffered in CdpSession.
    2. DOM mutation rate: < 3 mutations/sec over dom_settle_ms window.
       Injects MutationObserver JS snippet via CDP Runtime.evaluate.

    page_type overrides:
    - "static": returns after 200ms with stable=True
    - "ssr":    network quiescence only, DOM settle skipped
    - "spa":    both signals
    - "streaming": DOM tolerance raised to <10/sec, network signal skipped

    Returns:
        WaitForStableResult(stable, wait_duration_ms, signals)
        Always returns (never raises) — on timeout returns stable=False with warning.
    """
```

### 3.5 CDP Session (`cdp/session.py`)

```python
class CdpSession:
    """
    Manages a persistent Chrome DevTools Protocol WebSocket session.

    Responsibilities (per spec §7.7):
    - Network domain: buffer all requests/responses via event listeners
    - Runtime domain: execute JS snippets (MutationObserver, axe-core)
    - Console domain: buffer console.error and console.warn messages

    Connection:
    - Connects to ws://{cdp_host}:{cdp_port} (the sandbox's exposed CDP port)
    - Uses websockets library (pure Python, no Node.js dependency)

    Error handling:
    - On connection drop: auto-reconnects up to 3 times with 1s delay
    - On timeout: raises CdpTimeoutError (caller decides what to do)
    - Buffers are preserved across reconnections

    Phase 1: implements Network + Console domains.
    Phase 2: adds Runtime domain for DOM mutation observation.
    """

    def __init__(self, cdp_url: str) -> None: ...

    async def connect(self) -> None: ...
    async def close(self) -> None: ...

    async def enable_network_capture(self) -> None:
        """Enable CDP Network domain, start buffering requests."""

    async def enable_console_capture(self) -> None:
        """Enable CDP Log/Console domain, start buffering messages."""

    @property
    def pending_request_count(self) -> int:
        """Number of in-flight XHR/fetch requests. Used by wait_for_stable."""

    @property
    def console_messages(self) -> list[ConsoleMessage]:
        """All captured console messages since session start."""

    async def evaluate_js(self, expression: str, *, timeout_ms: int = 5000) -> Any:
        """Execute JS in the page context. Returns the result or raises CdpTimeoutError."""
```

## 6. `runner/run_manager.py`

```python
class RunManager:
    """
    Orchestrates a full test run:
    1. Loads test case
    2. Spawns (or connects to) sandbox container
    3. Initializes MCP client and CDP session
    4. Builds the agent graph and initial state
    5. Invokes graph.invoke(initial_state)
    6. Tears down sandbox
    7. Returns RunResult

    This is the integration seam — it owns the resource lifecycle.
    Guarantees: even if the graph raises, sandbox is stopped and resources closed.
    """

    def __init__(
        self,
        *,
        sandbox_manager: SandboxManager,
        sandbox_image: str = "project-hawkeye-hawkeye-sandbox:latest",
        model: str = "claude-sonnet-4-20250514",
    ) -> None: ...

    async def run(
        self,
        test_case: TestCase,
        *,
        browser_override: str | None = None,
        max_steps_override: int | None = None,
        timeout_override: int | None = None,
        output_dir: Path | None = None,
    ) -> RunResult:
        """
        Full end-to-end run. Handles all setup and teardown.

        On any unhandled exception: logs the error, stops the sandbox,
        and returns RunResult(status="errored", ...) — never propagates.
        """
```

```python
        "input_per_mtok": 3.00,
        "output_per_mtok": 15.00,
        "cache_read_per_mtok": 0.30,

    },
    "claude-opus-4-20250514": {
        "input_per_mtok": 15.00,
        "output_per_mtok": 75.00,
        "cache_read_per_mtok": 1.50,
        "cache_write_per_mtok": 18.75,
    },
    "claude-haiku-4-5-20251001": {
        "input_per_mtok": 0.80,
        "output_per_mtok": 4.00,
        "cache_read_per_mtok": 0.08,
        "cache_write_per_mtok": 1.00,
    },

def compute_step_cost(model: str, usage: dict) -> float:
    """
    Per spec §10.3.
    Falls back to most expensive known model pricing if model not in table,
    and logs a WARNING.
    """
    pricing = MODEL_PRICING.get(model)
    if pricing is None:
        import warnings
        warnings.warn(f"Unknown model {model!r} — using Opus pricing as conservative estimate")
        pricing = MODEL_PRICING["claude-opus-4-20250514"]
    return (
        usage.get("input_tokens", 0) * pricing["input_per_mtok"] / 1_000_000
        + usage.get("output_tokens", 0) * pricing["output_per_mtok"] / 1_000_000
        + usage.get("cache_read_input_tokens", 0) * pricing["cache_read_per_mtok"] / 1_000_000
        + usage.get("cache_creation_input_tokens", 0) * pricing["cache_write_per_mtok"] / 1_000_000
    )
```

## 12. Testing Strategy

### 12.1 Unit Tests (`tests/unit/`)

**`test_yaml_loader.py`**

Tests the full validation matrix:
- Valid minimal test case (only required fields) parses correctly with all defaults applied
- Valid full test case (all fields) round-trips correctly
- Missing `id` field raises `TestCaseValidationError` with field name in message
- Missing `target.url` raises `TestCaseValidationError`
- Missing `goal` raises `TestCaseValidationError`
- Invalid priority value (`"P5"`) raises `TestCaseValidationError`
- Invalid navigation_policy value raises `TestCaseValidationError`
- Malformed YAML (syntax error) raises `yaml.YAMLError`
- Non-existent file raises `FileNotFoundError`
- JSON file (not YAML) parses correctly
- Test case with no `steps` sets `steps=None` and agent can still run
- Test case with empty `assertions` list runs without assertion evaluation
- Constraints defaults: `max_steps=30`, `timeout_seconds=180` when absent

**`test_prompt_builder.py`**

- Full test case produces prompt containing all checkpoints
- No-steps test case produces prompt without the guided steps section
- Navigation policy "interact_only" produces the correct rule text (verbatim from spec §5.2)
- Hints are rendered as bullet points
- Forbidden actions are rendered as a bullet list
- Prompt is deterministic (same input → same output, 100 calls)

**`test_trace_collector.py`**

- Empty trace produces zero-value summary with no errors
- `compute_step_cost` returns 0.0 for zero tokens
- `compute_step_cost` with known values matches hand-calculated result
- Unknown model falls back to Opus pricing and logs a warning
- `get_summary()` percentiles correct for small datasets (p50, p95, p99)
- `get_summary()` handles single-trace case (no division by zero)
- `write_json()` produces valid JSON matching expected schema

**`test_edges.py`**

Tests the four routing functions against constructed `AgentState` dicts:

- `route_after_reason`:
  - `stop_reason="tool_use"` and no error → `"act"`
  - `stop_reason="end_turn"` → `"goal_check"`
  - `stop_reason="max_tokens"` → `"goal_check"`
  - `last_error` set with reason-origin → `"error"`
  - `status="timed_out"` → `"goal_check"` (forces finalize path)

- `route_after_act`:
  - `consecutive_errors=0` → `"observe"`
  - `consecutive_errors=3` → `"error"`
  - `status="timed_out"` → `"error"`

- `route_after_goal_check`:
  - `goal_complete=True` → `"finalize"`
  - `status="errored"` → `"finalize"`
  - `goal_complete=False`, `status="running"` → `"observe"`

- `route_after_error`:
  - `status="errored"` → `"finalize"`
  - `status="running"`, `consecutive_errors=1` → `"observe"`

**`test_assert_text_present.py`**

- Exact text found → `passed=True`
- Exact text not found → `passed=False`
- Regex pattern matches → `passed=True`
- Regex pattern doesn't match → `passed=False`
- Case-sensitive match fails with wrong case
- Case-insensitive match succeeds with wrong case
- Empty snapshot → `passed=False` (not a crash)
- Invalid regex pattern → `passed=False` with error message in details (not a crash)

**`test_models.py`**

- `StepTrace` default values are correct
- `ErrorInfo` `recoverable` defaults to `True`
- `RunStatus` enum values match expected strings
- `AgentState` TypedDict can be constructed with all required fields

### 12.2 Integration Tests (`tests/integration/`)

These require a running sandbox container and are marked with `@pytest.mark.integration`.

**`test_mcp_client.py`**

- `PlaywrightMcpClient.initialize()` succeeds when sandbox is running
- `list_tools()` returns at least the 10 allowlisted Playwright tools
- `call_tool("browser_navigate", {"url": "https://example.com"})` succeeds
- `call_tool("browser_snapshot", {})` returns non-empty accessibility tree text
- `call_tool("browser_click", {"element": "nonexistent_ref"})` returns `is_error=True` (not a crash)
- Client reconnects after deliberate close of subprocess and retry

## 14. Critical Architectural Decisions and Rationale

### Decision 1: Python LangGraph (not TypeScript)

The spec §7.1 lists Node.js/TypeScript as the orchestrator runtime. However, CLAUDE.md unambiguously states "Orchestrator: Python 3.11+, LangGraph (Python), LangChain" and lists `langchain-ollama` in Phase 1 dependencies. The existing PoC (`test_mcp_wiring.py`) is Python. The `hawkeye_sandbox` package is Python. All compiled bytecode in `agent/` is CPython. This plan follows CLAUDE.md — Python throughout, TypeScript code in spec §7.3 is treated as pseudocode for the state machine design.

### Decision 2: `stdio` Transport for MCP (Phase 1)

The sandbox currently maps CDP port to a random host port (e.g., `127.0.0.1:49155`). There is no HTTP MCP server exposed on port 3100 at the host level. The entrypoint script in spec §4.3 shows `npx @playwright/mcp@latest --port ${MCP_PORT} --headed` inside the container — this only binds to the container's internal port. Phase 1 uses `stdio` transport (spawning `npx @playwright/mcp --cdp-endpoint {cdp_url}` as a subprocess), exactly as in `test_mcp_wiring.py`. This avoids needing Docker bridge network changes until Phase 3.

### Decision 3: Phase 1 `wait_for_stable` uses CDP via host-mapped port

The sandbox exposes CDP via a socat proxy on port 9223 (not 9222 directly). `SandboxHandle.cdp_url` points to the host-mapped port. `CdpSession` connects directly to this URL. Phase 3 switches to container-internal addresses on the bridge network.

### Decision 4: No PostgreSQL in Phase 1

The spec §8 defines a full PostgreSQL schema. CLAUDE.md Phase 1 does not mention database. Phase 1 persists trace data to JSON files in `output_dir`. Database integration is Phase 2.

### Decision 5: Assertion Engine is non-fatal for unsupported types

Assertions of type `visual`, `state`, `network`, `accessibility`, and `performance` are Phase 2. Rather than failing the test with "not implemented," the assertion engine logs a `WARNING` and marks them as `"skipped"` — the run can still `PASS` or `FAIL` based on the implemented assertion types. This allows the Phase 1 YAML format to include future assertion types without breaking the runner.

### Decision 6: Context window management

The spec §10.5 describes auto-compaction after 10 steps. Phase 1 implements a simpler version: if `input_tokens` in the last step exceeds 150,000 (75% of 200K Sonnet context window), the `reason` node trims `messages` to keep only the system message, the last 10 tool call pairs, and a summary injected as a user turn. Full LangGraph `SqliteSaver` checkpointing (for crash recovery and time-travel debugging) is Phase 2.

## 15. Graceful Failure and Recovery Specification

Every component is designed to fail softly rather than crash the process:

1. **`PlaywrightMcpClient.call_tool()`**: Never raises to the caller. Returns `McpToolResult(content="", is_error=True, error_message=...)` on any failure. The `act` node checks `is_error` and increments `consecutive_errors`.

2. **`CdpSession.evaluate_js()`**: Raises `CdpTimeoutError` on timeout (the only exception type it propagates). All callers catch this and fall back to a safe default (e.g., `wait_for_stable` uses `stable=False` with `wait_duration_ms=timeout_ms`).

3. **`AssertionEngine.evaluate()`**: Wraps each assertion in `try/except Exception` — a buggy assertion never fails the whole run.

4. **`RunManager.run()`**: Uses `try/finally` to guarantee `sandbox.stop()` and `mcp_client.close()` are always called, even if the graph raises an unhandled exception.

5. **`observe_node()`**: If `browser_snapshot` fails 5 times consecutively, sets `status="errored"` and returns immediately (routes to error_handler, then finalize). The process does not hang.

6. **`reason_node()`**: Implements its own retry loop for 429/500 errors before delegating to error_handler. The exponential backoff is `[2, 4, 8]` seconds — aggressive enough to survive burst rate limits without hanging indefinitely.

7. **`finalize_node()`**: Always runs, even if `status` is already `errored`. It writes whatever trace data was collected and prints the summary. Evidence is never silently lost.
